# 08. Mantenimiento, Auditoría y Eficiencia

## ?? Limpieza de repositorios
- Elimina ramas obsoletas: `git branch -d rama`
- Borra archivos innecesarios y actualiza `.gitignore`
- Usa `git gc` para optimizar el repositorio

## ?? Auditoría de cambios
- Revisa el historial con `git log`, `git blame`, `git diff`
- Usa `git bisect` para encontrar errores
- Configura auditoría en GitHub con logs y revisiones

## ? Optimización y troubleshooting
- Soluciona problemas de merge y conflictos
- Usa `git fsck` para verificar integridad
- Analiza el rendimiento con `git status`, `git count-objects`

## ?? Ejercicio Práctico
1. Elimina ramas y archivos innecesarios
2. Audita el historial de cambios en tu proyecto
3. Ejecuta comandos de optimización y verifica el estado del repositorio
